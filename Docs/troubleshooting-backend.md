# Backend Troubleshooting

FastAPI app lives in `backend/app/`. In production it runs on a Python web
service; locally it is started with Uvicorn.

---

## 1. Quick health probes

| Endpoint | What it checks | Healthy response |
|----------|----------------|------------------|
| `GET /` | Process is up | `{"name":"HealthAI Ethics Assistant API", ...}` |
| `GET /health` | App is responding | `{"status":"ok"}` |
| `GET /health/db` | Engine can run `SELECT 1` | `{"status":"ok","backend":"postgresql"}` |
| `GET /docs` | OpenAPI / Swagger UI | Renders interactive docs |

```bash
BASE=https://your-backend.example   # or http://localhost:8000
curl -i $BASE/health
curl -i $BASE/health/db
```

If `/health` is OK but `/health/db` returns **503**, jump to
[troubleshooting-database.md](troubleshooting-database.md).

## 2. Local backend not starting

### Symptom: `ModuleNotFoundError` / wrong Python
```bash
cd backend
python --version          # must be 3.11.x
source venv/bin/activate  # always activate before running uvicorn / pytest
pip install -r requirements.txt
```

### Symptom: `pydantic_core._pydantic_core.ValidationError` for settings
- `CORS_ORIGINS` in `.env` must be **valid JSON**, e.g.
  ```env
  CORS_ORIGINS=["http://localhost:5173"]
  ```
  not a bare string.
- `ACCESS_TOKEN_EXPIRE_MINUTES` must be an integer.

### Symptom: app crashes on import with DB error
Tables are created at startup via `Base.metadata.create_all(bind=engine)`.
If `DATABASE_URL` is unreachable, import itself fails. Either fix the URL or
fall back to SQLite for local dev:
```env
DATABASE_URL=sqlite:///./app.db
```

### Run the dev server
```bash
cd backend && source venv/bin/activate
uvicorn app.main:app --reload --port 8000
```

## 3. Backend deploy is failing

1. Check your provider's build and deploy logs.
2. Build command: `pip install -r requirements.txt` (set in `render.yaml`).
3. Start command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`.
4. Common causes:
   - Missing env var (`GEMINI_API_KEY`, `SECRET_KEY`, OAuth client IDs).
   - `DATABASE_URL` not linked to the Postgres instance.
   - Python version mismatch — confirm `runtime.txt` / `PYTHON_VERSION` env var.
5. Force a clean redeploy if you suspect a stale image or dependency cache.

## 4. CORS errors in the browser

```
Access to XMLHttpRequest at 'https://your-backend.example/...' from origin 'https://your-frontend.example'
has been blocked by CORS policy.
```

- Edit `CORS_ORIGINS` env var to include the exact frontend origin
  (no trailing slash, must match scheme):
  ```
  ["https://your-frontend.example"]
  ```
- Restart the backend service after saving.
- Verify with:
  ```bash
  curl -i -H "Origin: https://your-frontend.example" \
       -H "Access-Control-Request-Method: POST" \
       -X OPTIONS $BASE/ethics/chat
  ```
  The response must include `access-control-allow-origin: https://your-frontend.example`.

## 5. OAuth (Google / GitHub) not redirecting back

- `BACKEND_URL` env var must be the public backend URL, not `localhost`.
- `FRONTEND_URL` env var must be the public frontend URL.
- Provider console (Google Cloud / GitHub OAuth App) must whitelist:
  - Authorized redirect URI: `${BACKEND_URL}/auth/google/callback` (and
    `/auth/github/callback`).
- After updating env vars, redeploy.
- If callback returns to login page silently, check backend logs for the
  exception around `/auth/<provider>/callback`.

## 6. LLM (Gemini) failures

Gemini calls live in `backend/app/services/llm_service.py`.

```bash
# Quick smoke test (uses your local .env)
cd backend && source venv/bin/activate
python scripts/verify_gemini.py
```

Symptoms and fixes:

| Error | Likely cause |
|-------|--------------|
| `401 / API key not valid` | `GEMINI_API_KEY` missing or wrong; rotate at aistudio.google.com |
| `429` quota exceeded | Free tier RPM cap; wait or switch model |
| `model not found` | `GEMINI_MODEL` typo (current default `gemini-2.5-flash`) |
| Long hangs / timeouts | Cold start on Render free plan; first request after idle takes 30–60s |

Streaming chat path (`/chat/sessions/{id}/stream`): if the SSE stream stalls,
test it locally with `scripts/verify_chat_stream.py`.

## 7. Auth / JWT issues

- Token is stored in `localStorage('access_token')` by the frontend.
- Backend signs with `SECRET_KEY` (generated by Render the first time;
  changing it invalidates every existing token).
- 401 on `/auth/me` after login → check that the frontend is sending
  `Authorization: Bearer <token>` (handled by `frontend/src/api/index.js`).

## 8. Run tests locally

```bash
cd backend && source venv/bin/activate
python -m pytest -q
```

Tests force `DATABASE_URL=sqlite:///./test.db` via `tests/conftest.py`, so they
work without a real Postgres. If tests fail right after a refactor, re-check
that new fields on Pydantic schemas have defaults or are passed in fixtures.
